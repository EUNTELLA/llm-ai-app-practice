# 📈 결과 분석

## 🔹 1. 배치 사이즈 영향
- **결론: 배치 사이즈가 클수록 메모리 사용량은 정비례하여 증가함**
  - `배치 4`: 10.375 GB
  - `배치 8`: 10.808 GB
  - `배치 16`: 11.672 GB
- **원인**: 더 많은 데이터를 한 번에 처리하면서 중간 계산 값(활성화)을 더 많이 저장해야 하기 때문

## 🔹 2. Gradient Accumulation 효과
- **결론: 동일 연산량 대비, 메모리 사용량을 절약할 수 있음**
  - `실제 배치 16`: 11.672 GB
  - `배치 4 + 누적 4`: 10.375 GB (✅ **약 1.3 GB 절약**)
- **원인**: 그래디언트를 여러 스텝에 걸쳐 누적한 뒤 한 번에 업데이트하므로, 큰 배치를 사용하는 것과 같은 효과를 내면서도 메모리는 더 적게 사용함

## 🔹 3. Gradient Checkpointing 효과
- **결론: 중간 계산 값을 재계산하여 메모리를 절약함**
  - `배치 16 (적용 안함)`: 11.672 GB
  - `배치 16 (적용)`: 10.174 GB (✅ **약 1.5 GB 절약**)
- **원인**: 순전파 시 활성화 값을 모두 저장하지 않고, 역전파 시 필요할 때 동적으로 재계산하여 메모리 점유율을 낮춤

## 🔹 4. LoRA 효과
- **결론: 학습 대상 파라미터를 줄여 메모리를 획기적으로 절약함**
  - `Full-Tuning 최대 메모리`: ~11.672 GB
  - `LoRA 최대 메모리`: 4.252 GB (획기적 감소)
- **원인**: 전체 모델 파라미터(13억 개)가 아닌, 0.12%에 해당하는 일부(어댑터)만 학습시켜 그래디언트와 옵티마이저 관련 메모리 부담이 거의 없음

## 🔹 5. QLoRA 효과
- **결론: 양자화와 LoRA를 결합하여 최소한의 메모리로 튜닝을 가능하게 함**
  - `LoRA 최대 메모리`: 4.252 GB
  - `QLoRA 최대 메모리`: 2.419 GB (✅ **가장 효율적**)
- **주목할 점**: QLoRA 사용 시 필요한 메모리는 **원본(fp16) 모델을 로드하는 데 드는 메모리(2.481 GB)보다도 적음**
- **원인**: 모델 자체를 4비트로 양자화하여 용량을 줄이고, 그 위에 LoRA를 적용하여 학습에 필요한 메모리까지 최소화하는 극강의 효율을 보여줌

# 결론

### **가장 효율적인 튜닝 기법: QLoRA**
- **결론**: 제한된 단일 GPU 환경에서 대규모 모델을 튜닝하는 가장 강력하고 현실적인 방법은 **QLoRA**
- **핵심**: 모델 양자화(4bit)와 LoRA를 결합하여, **원본 모델을 로드하는 것보다도 적은 메모리(2.419 GB)**로 튜닝을 가능하게 함
- **의의**: 저사양 GPU에서도 대규모 언어 모델의 파인튜닝을 수행할 수 있는 길을 열어줌

### **기법별 효율성 비교**
1.  **QLoRA**: (가장 효율적) 최소한의 메모리로 튜닝 가능
2.  **LoRA**: (매우 효율적) 학습 파라미터를 크게 줄여 메모리 부담을 획기적으로 낮춤
3.  **Gradient Checkpointing / Accumulation**: Full-Tuning 시 메모리 절약에 도움이 되나, PEFT 기법의 효율에는 미치지 못함
4.  **Full Fine-Tuning**: (비효율적) 가장 많은 메모리를 요구

### **핵심 원리 요약**
- **PEFT (LoRA, QLoRA)**: '학습하는 파라미터 수'를 줄여 메모리를 절약
- **양자화 (Quantization)**: '모델 자체의 크기'를 줄여 메모리를 절약
- **Checkpointing / Accumulation**: '학습 과정에서의 메모리 사용 방식'을 최적화하여 절약
